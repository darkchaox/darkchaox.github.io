在Jekyll 3.0中手动添加pagination

发表于 2018-08-16 更新于 2021-03-16 分类于 技术 Valine：
本文字数： 409 阅读时长 ≈ 1 分钟

使用Hyde主题，在本地查看Blog时，文章无法显示；
启动转化和本地服务bundle exec jekyll serve时会出错：

Deprecation: You appear to have pagination turned on, but you haven’t included the jekyll-paginate gem. Ensure you have gems: [jekyll-paginate] in your configuration file.

因为Jekyll 3中并没有集成pagination，需要手动安装。
1.打开终端，输入gem install jekyll-paginate
2.打开项目根目录下的_config.yml，添加以下内容：

gems :

  • jekyll-paginate

保存后重新执行bundle exec jekyll serve即可。

关于InputStream的read方法返回值的理解

发表于 2018-08-16 更新于 2021-03-17 分类于 技术 Valine：
本文字数： 268 阅读时长 ≈ 1 分钟

  • 官方文档的解释是：

    Returns:the total number of bytes read into the buffer, or -1 if there is no more data because the end of the stream has been reached.

    返回值：返回的是缓冲流的总字节数，到达文件末尾时返回-1。

    当时很难理解，为什么既能返回总字节数，又可以返回-1，怎么能返回两个值…

    正确的理解是：
    当调用read方法，读取第一个字节时，如果为null（说明已到文件末尾），则返回为-1，
    否则，继续读取下一个字节，直到为null，此时就返回读取的字节总数。
